### Regression Discovered
When accessing `action=edit&feed_id=999` (non-existent ID), the system was showing "Create New Feed" form instead of an error message. This is a security issue as it allows ID manipulation attacks.

### Root Cause
In `templates/admin/feed-editor.php`, the code was checking if `$feed` was empty and setting `$is_new = true` without considering that the user explicitly requested to EDIT a specific feed ID. The fallback to "create" mode was incorrect behavior.

### Fix Applied
Added validation in `feed-editor.php` to check:
1. If action is 'edit'
2. AND feed_id > 0 (user specified an ID)
3. AND feed is empty (doesn't exist in database)
4. THEN show "Feed Not Found" error page

real Instagram data (URLs, captions, counts), but image display requires either:

2. Instagram official embed widgets
3. Caching images locally

The current implementation proves real data is being fetched - image proxying could
be added as a future enhancement.
